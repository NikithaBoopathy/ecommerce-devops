# Which Service Type To Use?

Kubernetes has multiple service types.

| Type         | Purpose                |
| ------------ | ---------------------- |
| ClusterIP    | Internal communication |
| NodePort     | Access from outside    |
| LoadBalancer | Cloud external access  |
| ExternalName | DNS mapping            |

# Step 1 Deployment YAML

Create:

```text id="1qjlwm"
user-deployment.yml
```

Example:

```yaml id="qk5bgp"
apiVersion: apps/v1
kind: Deployment

metadata:
  name: user-deployment

spec:
  replicas: 1

  selector:
    matchLabels:
      app: user-service

  template:
    metadata:
      labels:
        app: user-service

    spec:
      containers:
      - name: user-service

        image: nikithaboopathy/ecommerce-user-service:v1

        ports:
        - containerPort: 8081
```

---

# Understand This Properly

## replicas

```yaml id="fjlwm0"
replicas: 1
```

Means:

* create 1 pod

---

## selector

```yaml id="zjlwm1"
selector:
  matchLabels:
    app: user-service
```

Deployment manages pods with this label.

---

## template.labels

```yaml id="6xjlwm"
labels:
  app: user-service
```

Pod gets this label.

---

# MOST IMPORTANT CONCEPT

Selector matches labels.

```text id="tx0pva"
Deployment
   ↓ selector
app=user-service
   ↓
Pod with same label
```

Same concept later used in Services also.

---

# Step 2 Service YAML

Create:

```text id="0jlwmn"
user-service.yml
```

Example:

```yaml id="33tbxv"
apiVersion: v1
kind: Service

metadata:
  name: user-service

spec:
  type: NodePort

  selector:
    app: user-service

  ports:
    - protocol: TCP
      port: 8081
      targetPort: 8081
      nodePort: 30081
```

---

# Understand Service Properly

## selector

```yaml id="u9jlwm"
selector:
  app: user-service
```

Service forwards traffic to pods having:

* app=user-service

---

## port

Internal service port.

---

## targetPort

Pod container port.

Spring Boot running on:

```text id="7jlwmr"
8081
```

---

## nodePort

External VM port.

Access using:

```bash id="yzjlwm"
curl http://<VM-IP>:30081/users
```
